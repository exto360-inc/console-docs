Introduction ​

Exto Console is the central control plane for the Exto platform. It unifies the work that, in most B2B SaaS companies, is scattered across spreadsheets, ticketing systems, and side scripts — managing customers, provisioning tenant environments, controlling identity, orchestrating releases, and billing accurately for usage.

What the Console does ​

The Console is built around four pillars:

  1. Universal Identity & Access. A single sign-on experience for every Exto application. The Console binds users to the customers, tenants, and instances they are entitled to use, and enforces multi-factor authentication, password policy, and identity-provider rules for each customer organisation.

  2. Customer, Tenant & Instance Management. Customers represent the companies you sell to. Each customer owns one or more tenants — isolated logical environments inside a hosted instance. The Console handles the entire lifecycle: create, configure, suspend, archive, move between customers, or migrate to a different instance.

  3. Release Orchestration. Define a deployment pipeline once, then ship every release through it. Releases bundle service images and environment-variable changes, run through stages with approval and trigger gates, and produce a verifiable audit trail.

  4. Metering & Billing. Define meters for the things you charge for. Bind them to plans, assign plans to customers, and let the Console compute billed usage automatically — with first-class support for production vs. non-production environments, per-tenant overrides, and overage pricing.

Two surfaces, two audiences ​

The Console exposes two distinct workspaces:

  • The Admin Console is for Exto operators — infrastructure ops, account managers, QA admins, and platform admins. It exposes everything the platform can do: every customer, every tenant, every instance, every release.

  • The Customer Portal is what your customers see when they log in. It is scoped to a single customer, hides operator-only functionality, and is gated by portal roles (owner, admin, member).

Both surfaces share the same identity layer, the same release machinery, and the same billing engine — they just present different slices of it.

How to use this guide ​

If you are an Exto operator, start with Quick Start and then jump into the Admin Console section.

If you are a customer admin signing in for the first time, read Signing In and the Customer Portal Overview.

If you want to understand how the pieces fit together before opening the UI, read Core Concepts end-to-end. Everything else in the guide refers back to those concepts.
