Admin Console Overview ​

The Admin Console is the workspace for Exto operators. It exposes every customer, tenant, instance, release, and billing record in the platform, gated by your internal role and scope.

Sidebar map ​

The left-hand sidebar is organised by domain:

SectionWhat lives here
DashboardPlatform health and recent activity at a glance. See Dashboard.
CustomersThe companies you sell to. Create, suspend, churn, and manage. See Customers.
TenantsCustomer environments running on instances. See Tenants.
Tenant InsightsFleet-wide tenant usage analytics. See Tenant Insights.
InstancesHosted environments (clusters) that run tenants. See Instances.
MigrationsTenant clones, deletes, exports, imports, and instance replaces. See Migrations.
Internal UsersExto operators and their roles. See Internal Users.
User AttributesCustom profile fields for customer users. See User Attributes.
ReleasesNew releases and their pipeline status. See Releases.
Release ConfigurationGlobal pipeline, approval, and SLA settings. See Release Configuration.
ChangelogCustomer-facing release notes. See Changelog.
Environment VariablesGlobal variables and per-instance overrides. See Environment Variables.
FinancePlans, meters, customers, invoices, payments, usage. See Finance Overview.
Audit LogEvery action recorded against its actor. See Audit Log.
Email LogsOutbound notification history. See Email Logs.
System → WorkersBackground worker health. See System Workers.
System → DiscrepanciesTriage inbox for dangling tenant data. See Discrepancies.

What you can see ​

Your internal role and scope determine what is visible:

  • Platform Admins and Infra Ops see everything across the platform.
  • Account Managers see only customers in their scope. Other customers are hidden, not just locked.
  • QA Admins see only customers in their customer scope and may operate on instances in their instance scope.

See Roles & Permissions.

Common patterns ​

The Admin Console is consistent across pages. Once you have used a few list views, the rest is familiar:

  • List pages show a toolbar (search, filter dropdowns, Clear button) above a sortable table. The count to the right of the toolbar shows filtered vs. total rows.
  • Detail pages open from clicking a row. They have a header with status, a primary action set, and one or more tabs.
  • Destructive actions (suspend, churn, archive, decommission, delete) always open a confirmation modal — usually with a required reason field.
  • Status badges are colour-coded everywhere: green = healthy, orange = warning, red = problem, grey = retired.
  • Timestamps are relative on lists ("2 hours ago") and absolute on detail pages, with the full date available on hover.
  • Row actions appear as icon buttons in the rightmost column, sometimes on hover.

Focus mode ​

If you operate against a single customer for a while, click Focus on this customer on the customer's detail page. While focused, the Dashboard and other aggregate views narrow to just that customer's data. Exit focus from the same control.
